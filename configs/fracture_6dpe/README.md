# Формат датасета для 3D-пазла фрагментов объекта

Этот README описывает формат датасета, основанный на BOP, с дополнительной разметкой для задач:

- **2D/2.5D сегментации фрагментов** (instance segmentation);
- **оценки 6D-позы фрагмента в сцене/камере**;
- **оценки позы фрагмента в каноническом пространстве объекта** (3D-пазл);
- **shape completion** фрагмента (полная форма, включая невидимую часть);
- **оценки центра масс фрагмента**;
- **глобальной согласованности 3D-пазла** (сборка фрагментов в целый объект).

---

## 1. Системы координат

В датасете используются три системы координат:

- **O** — каноническая СК объекта  
  В ней задан файл `obj_000001.ply` (целый объект до разрушения).

- **C** — СК камеры / сцены  
  В ней заданы `cam_K`, `cam_R_m2c`, `cam_t_m2c`, и в ней живут `rgb`, `depth`.

- **F** — локальная СК фрагмента  
  Для каждого фрагмента определяется отдельно:
  - начало координат — центр его 3D-bbox (в канонической форме фрагмента);
  - в этой СК удобно хранить:
    - полную форму фрагмента (его меш в `fragments/`);
    - центр масс фрагмента `com_F`.

Переход между СК:

- из F в объект O:  
  \[
    x_O = R_{O\leftarrow F} \, x_F + t_{O\leftarrow F}
  \]
  где `R_O_from_F`, `t_O_from_F` заданы в `fragments_gt.json`;
- из O и F в камеру C — через `cam_R_m2c`, `cam_t_m2c` в `scene_gt.json`  
  (в генераторе задаётся однозначно).

## 2. Структура директорий

```text
dataset_root/
├── camera.json                 # (опционально) глобальные параметры камеры
├── dataset_info.json           # (опционально) мета-информация о датасете
├── models_priori_objects/
│   ├── models_info.json        # BOP: информация о моделях по obj_id
│   └── obj_000001.ply          # каноническая 3D-модель исходного объекта (СК O)
├── train/
│   ├── rgb/                    # RGB-кадры
│   ├── depth/                  # карты глубины (PNG, uint16)
│   ├── mask/                   # маски силуэтов фрагментов
│   ├── mask_visib/             # маски видимой части фрагментов
│   ├── scene_camera.json       # параметры камеры для каждого кадра
│   ├── scene_gt.json           # позы фрагментов в камере (C), BOP-формат
│   ├── scene_gt_info.json      # доп. инфа по GT (bbox, видимость и т.п.)
│   ├── scene_gt_coco.json      # аннотации в формате COCO
│   ├── fragments_gt.json       # формы и позы фрагментов в каноне (F → O, CoM)
│   ├── fragments/              # меши фрагментов (STL/PLY) в их локальных СК F
│   └── scenes.json             # (опц.) связывает scene_id ↔ image_ids
├── val/
│   └── ...                     # то же, что и train/
└── test/
    └── ...                     # то же, при необходимости с GT
```


## Структура `fragments_gt.json`
```
{
  "scene_ids":    [...],
  "frag_counts":  [...],
  "total_frags":  N,

  "frag_scene_id": [...],
  "frag_local_idx": [...],

  "com_F":       [...],
  "R_O_from_F":  [...],
  "t_O_from_F":  [...]
  "mesh_filenames": [
  "obj_000001_cell_1.ply",
  "obj_000001_cell_2.ply",
  ...
]
}
```

## Структура `fragments/`
```
train/fragments/
  obj_000001_cell_1.ply
  obj_000001_cell_2.ply
  ...
```

## Модифицируем структуру `scene_gt.json`

Добавляем `frag_local_idx`
```
"im_id": [
  {
    "obj_id": 1,
    "frag_local_idx": 1,
    "scene_id": 1,
    "cam_R_m2c": [...],
    "cam_t_m2c": [...]
  },
  {
    "obj_id": 1,
    "frag_local_idx": 2,
    "scene_id": 1,
    "cam_R_m2c": [...],
    "cam_t_m2c": [...]
  }
]
```

## Структура `scenes.json`

```
{
  "scene_ids": [1, 2],

  "scenes": {
    "1": {
      "scene_id": 1,
      "obj_id": 1,                                   // BOP-obj_id исходного объекта
      "model_file": "models_priori_objects/obj_000001.ply",

      "frag_count": 4,                               // сколько фрагментов в этом пазле
      "frag_local_indices": [1, 2, 3, 4],           // локальные индексы фрагментов (1..K)

      "image_ids": [0, 3, 7, 9]                     // кадры, где встречаются фрагменты этого объекта
    },

    "2": {
      "scene_id": 2,
      "obj_id": 2,
      "model_file": "models_priori_objects/obj_000002.ply",

      "frag_count": 5,
      "frag_local_indices": [1, 2, 3, 4, 5],

      "image_ids": [1, 2, 5, 8]
    }
  }
}
```
